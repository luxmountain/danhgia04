### Test 3: Chat với Intent Detection

Nếu câu hỏi **không** trong Knowledge Base, chatbot sẽ:

```
"recommend sản phẩm cho tôi"
  → Intent: recommend
  → Trả lời: Sản phẩm từ lịch sử mua hàng

"tôi muốn sản phẩm giá rẻ"
  → Intent: cheap
  → Trả lời: Sản phẩm giá rẻ nhất

"show me electronics"
  → Intent: category
  → Trả lời: Sản phẩm trong Electronics

"sản phẩm bán chạy nhất"
  → Intent: best
  → Trả lời: Top sản phẩm bán chạy
```

## 📊 FLOW CHATBOT

```
User sends message
    ↓
Check Knowledge Base first
    ├─ YES: Return KB answer
    │   ├─ KB suggest best? → Add top 5 products
    │   ├─ KB suggest cheap? → Add 5 cheap products
    │   └─ KB no products? → Only answer
    │
    └─ NO: Use Intent Detection
        ├─ recommend → User history products
        ├─ cheap → Cheap products
        ├─ best → Popular products
        ├─ category → Category products
        └─ default → Search results
```

## 🔍 KIỂM TRA LOGIC

### Knowledge Base vs Intent Detection

**Ưu tiên:**
1. **Knowledge Base TRƯỚC** - Nếu câu hỏi match KB → Trả lời từ KB
2. **Intent Detection SAU** - Nếu không match KB → Dùng intent

**Ví dụ:**

```
User: "các danh mục sản phẩm là gì"
  ✓ Match KB → Trả lời: "Chúng tôi có 10 danh mục..."
  ✓ Không cần search products

User: "recommend sản phẩm cho tôi"
  ✓ Not in KB → Intent detection
  ✓ Intent: recommend
  ✓ Trả lời: "Dựa trên lịch sử..."
  ✓ Show 5 products from history

User: "hello"
  ✓ Not in KB
  ✓ No intent match
  ✓ Default: "Những sản phẩm bạn có thể quan tâm"
  ✓ Show best products
```

### Page Routes
```
GET  /                    → Home page
GET  /category/<name>     → Category page (e.g., /category/Electronics)
```

### API Routes
```
GET  /api/categories      → List all categories
GET  /api/products        → Get products (with sort/category params)

POST /api/chat            → Chat with KB + Intent detection
```
